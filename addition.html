Definition: Add curly braces { } around the body of a control structure (e.g., if, for, while) to make the scope explicit and reduce ambiguity.


- public boolean matchesSafely(final Result item) {
-     for(Failure f: item.getFailures())
-     {
-         return CoreMatchers.instanceOf(causeClass).matches(f.getException().getCause());
-     }
-     return false;
- }
+ public boolean matchesSafely(Result item) {
+     for(Failure f: item.getFailures()) {
+         return CoreMatchers.instanceOf(causeClass).matches(f.getException().getCause());
+     }
+     return false;
+ }

Rationale: Adding braces makes control flow clearer, prevents errors when adding new statements, and improves code readability and maintainability.

Definition: Replace a direct call to the superclass constructor with a delegating call to another constructor in the same class, which in turn invokes the superclass constructor.


-    public Http11Nio2Protocol() {
-        super(new Nio2Endpoint());
-    }
+    public Http11Nio2Protocol() {
+        this(new Nio2Endpoint());
+    }

Rationale: Centralizes constructor logic, reducing duplication and ensuring consistent initialization. Delegation improves maintainability by routing all constructor invocations through a single entry point.

Definition: Consolidate and parameterize hardcoded documentation links into reusable, maintainable variables to reduce duplication and improve consistency.


-    docs-spring: "{docs-site}/spring-framework/docs/{spring-version}" 
-    docs-spring-framework: '{docs-site}/spring-framework/docs/{spring-version}'
-    api-spring-framework: '{docs-spring-framework}/javadoc-api/org/springframework'
-    docs-graalvm: 'https://www.graalvm.org/22.3/reference-manual'
-    docs-spring-boot: '{docs-site}/spring-boot/docs/current/reference'
+    spring-framework-docs-root: '{docs-site}/spring-framework/docs'
+    spring-framework-api: '{spring-framework-docs-root}/{spring-version}/javadoc-api/org/springframework'
+    spring-framework-api-kdoc: '{spring-framework-docs-root}/{spring-version}/kdoc-api'
+    spring-framework-reference: '{spring-framework-docs-root}/{spring-version}/reference'

Rationale: Reduces repetition by centralizing the root documentation path into a single variable. This makes future updates easier (e.g., changing the root URL once instead of in multiple places), improves readability, and ensures consistency across different documentation references.
