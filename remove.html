Definition: Remove inline debug logging notes from parameter validation messages, keeping only essential error messages.


 parameters.copyFail=Vytvoření kopie původních hodnot parametrů pro účely DEBUG logování selhalo
 parameters.decodeFail.debug=Dekódování znaku selhalo. Parametr [{0}] s hodnotou [{1}] byl ignorován.
-parameters.fallToDebug=\n\
-\ Poznámka: další výskyty chyb parametrů budou zalogovány v DEBUG úrovni.
-parameters.maxCountFail.fallToDebug=\n\
-\ Poznámka: další výskyty této chyby budou zalogovány v úrovni DEBUG.

Rationale: Removes redundant debug-level logging notes from resource bundles, reducing noise in user-facing messages while keeping logging behavior intact at the DEBUG level. This improves readability of localized error messages and prevents unnecessary duplication.

Definition: Remove unnecessary braces around a single-statement for loop body to simplify the code.


- for(Failure f: item.getFailures())
- {
-     return CoreMatchers.instanceOf(causeClass).matches(f.getException().getCause());
- }
+ for(Failure f: item.getFailures())
+     return CoreMatchers.instanceOf(causeClass).matches(f.getException().getCause());

Rationale: Since the loop body contains only a single statement, removing the braces improves readability by reducing visual clutter. It follows common Java style conventions where braces are optional for one-line blocks, though some teams may prefer always using braces for consistency and to prevent future errors.
